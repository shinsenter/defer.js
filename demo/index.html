Get Started:

See how easy it is to use Defer.js on your site!

Quick Installation:

Just add a <script> tag linking to the library right after your opening <head> tag. The file defer.min.js is very small, and you can even inline it to avoid an extra HTTP request.

Fully Defer <script> Tags:

Slow scripts (like third-party add-ons) can affect site performance.
Fully deferring <script> tags can help avoid Web Vitals issues.

Using the special attribute type="deferjs":

You can easily fully defer any script tag by setting its type attribute to deferjs.
This trick also works perfectly with <script src> tags.
Note: Since version 3.0, scripts using type="deferjs" only run after user interaction if Defer.lazy=true.
Note: Resource hints feature was added since v3.2 as it is recommended to prevent issues called Taming the Waterfall. This feature is discussed at #112.

Using your own attribute value, like type="my-magic":

If you prefer not to use the type="deferjs" attribute, you can choose your own value.

Run After User Interaction:

Sometimes, you may want your code to run only when there is user activity.
The third argument tells Defer() to delay the function and wait until the user starts interacting with your page.

The Defer.lazy() Method

Setting Defer.lazy=true tells the library to delay the execution of deferred scripts when the user starts interacting with the page, regardless of the page load event.
It will override the waitForUserAction argument in the Defer() method.

Defer.dom([selector], [delay], [unveiledClass], [resolver], [observeOptions])

The Defer.dom() method is useful in the following use cases:
  • Lazy loading images, media, iframe tags, etc. on your website.
  • Preventing the download of third-party libraries or add-ons unless they are needed.
  • Scroll-reveal features, such as handling AJAX updates when a block enters the viewport.

Basic Usage:

Browsers load images when they see the src attribute, even if images are not visible.
To defer loading, put the image URL in a different attribute like data-src.
Sometimes, the src attribute can load a tiny placeholder image first.

Lazy Loading Responsive Images with data-srcset and data-sizes:

Using the srcset attribute has made responsive image sizing much simpler. It allows you to define a list of differently-sized versions of the same image and provide information about the size of each one. Then, the client (browser) gets to decide which one to use.
We can use the same trick as the above examples. We specify an image URL set in the data-srcset attribute of the image tag.

Lazy Loading Responsive Images with Different Formats:

Different browsers support different image formats. We might want to send a fancy new image format such as WebP to browsers that can render it, and fall back to trusty old JPEGs in browsers that can't.
We can also use the same trick as the above examples for picture tags and their child HTML nodes.

Lazy Loading animate.css:

In this example, we set waitForUserAction=5000 (the 5th parameter of the Defer.css function). This means the website will wait until there is user interaction before starting to load the animate.css library. If more than 5 seconds pass without any user interaction, the library will still be loaded, and the scripts will execute.
When the Animate.css library is downloaded, we will add CSS classes from Animate.css to every tag with class="demo" on this page. However, no tag will be animated unless you scroll to its position.

We use the Defer.js() method to defer loading 3rd-party JavaScript libraries, add-ons, etc., without blocking the page rendering.
